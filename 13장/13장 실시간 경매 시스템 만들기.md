# 서버센트 이벤트

* npm i sse
    * sse + socket.io 모듈 사용 
* 모든 사람이 같은 시간에 경매가 종료되어야 함

* 모든 사람에게 같은 시간이 표시되어야 함

* 클라이언트 시간은 믿을 수 없음(조작 가능)

* 따라서 서버 시간을 주기적으로 클라이언트로 내려보내줌

* 이 때 서버에서 클라이언트로 단방향 통신을 하기 때문에 서버센트 이벤트(Server Sent Events, SSE)가 적합

* 웹 소켓은 실시간으로 입찰할 때 사용
